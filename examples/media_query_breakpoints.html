Media Query Breakpoints Guide

A comprehensive guide to using media queries effectively for responsive web design.

Current viewport width: px

What Are Media Queries?

Media queries allow you to apply CSS styles based on device characteristics such as screen width, height, orientation, and more. They're a fundamental building block for responsive design.

@media (min-width: 768px) { /* These styles apply when the viewport is at least 768px wide */ .container { width: 750px; } }

Basic Syntax

@media [media-type] ([media-feature]) { /* CSS rules to apply when media query matches */ }

Common media types include:

  • screen: Computers, tablets, phones
  • print: Print preview mode and printed pages
  • all: All media types (default)

Common media features include:

  • width, min-width, max-width: Viewport width
  • height, min-height, max-height: Viewport height
  • orientation: Portrait or landscape
  • aspect-ratio: Width-to-height ratio
  • resolution: Pixel density

Standard Breakpoints

While you can set breakpoints anywhere, the following are commonly used in frameworks like Bootstrap and responsive designs:

Extra Small (xs) < 576px Portrait phones Default styles (no media query)

Medium (md) ≥ 768px Tablets @media (min-width: 768px) { ... }

Mobile-First vs. Desktop-First

Mobile-First Approach

Start with styles for mobile devices and add complexity as the screen gets larger using min-width queries.

/* Base styles for mobile */ .container { width: 100%; } /* Tablet and larger */ @media (min-width: 768px) { .container { width: 750px; } } /* Desktop and larger */ @media (min-width: 992px) { .container { width: 970px; } }

Pros: Prioritizes mobile experience, results in cleaner code, better performance for mobile.

Desktop-First Approach

Start with styles for desktop devices and scale down using max-width queries.

/* Base styles for desktop */ .container { width: 1170px; } /* Tablet and smaller */ @media (max-width: 991px) { .container { width: 750px; } } /* Mobile and smaller */ @media (max-width: 767px) { .container { width: 100%; } }

Pros: Easier if you have an existing desktop-focused design, can be simpler for complex desktop layouts.

Best Practices

  • Mobile-first is recommended for most modern websites
  • Use min-width for mobile-first, max-width for desktop-first
  • Avoid mixing both approaches to prevent conflicts
  • Always include the viewport meta tag: <meta name="viewport" content="width=device-width, initial-scale=1.0">

Other Types of Media Queries

Media queries can target more than just viewport width. Here are some other useful types:

Width & Height

/* Viewport width */ @media (min-width: 768px) and (max-width: 1200px) { /* Applied between 768px and 1200px */ } /* Viewport height */ @media (min-height: 600px) { /* Applied when viewport height is at least 600px */ }

Orientation

/* Portrait mode (height > width) */ @media (orientation: portrait) { .sidebar { display: none; } } /* Landscape mode (width > height) */ @media (orientation: landscape) { .sidebar { display: block; } }

Aspect Ratio

/* Widescreen displays (16:9 or wider) */ @media (min-aspect-ratio: 16/9) { .hero { height: 70vh; } } /* Standard displays (4:3 or similar) */ @media (aspect-ratio: 4/3) { .hero { height: 50vh; } }

Print Media

/* Styles for print */ @media print { .navigation, .footer, .sidebar { display: none; } body { font-size: 12pt; color: black; background: white; } a::after { content: " (" attr(href) ")"; } }

Feature Queries (@supports)

Feature queries let you apply styles based on browser support for specific CSS properties:

/* Apply styles only if CSS Grid is supported */ @supports (display: grid) { .container { display: grid; grid-template-columns: repeat(3, 1fr); } } /* Fallback if Grid is not supported */ @supports not (display: grid) { .container { display: flex; flex-wrap: wrap; } .item { width: 33.33%; } }

/* Base (mobile) typography */ .responsive-text h3 { font-size: 1.5rem; } .responsive-text p { font-size: 1rem; } /* Medium screens */ @media (min-width: 768px) { .responsive-text h3 { font-size: 1.8rem; } .responsive-text p { font-size: 1.1rem; } } /* Large screens */ @media (min-width: 992px) { .responsive-text h3 { font-size: 2rem; } .responsive-text p { font-size: 1.2rem; } }

Best Practices for Media Queries

  • Use relative units (em, rem, %) instead of px for more flexible layouts
  • Limit the number of breakpoints to those you actually need
  • Test on real devices, not just by resizing your browser window
  • Consider content when choosing breakpoints, not just device sizes
  • Keep your media queries organized, either grouped by component or by breakpoint
  • Use logical operators to create complex queries when needed
  • Combine media queries with CSS variables for more maintainable code
/* Complex media query with logical operators */ @media (min-width: 768px) and (max-width: 1024px) and (orientation: landscape) { /* Applied only on tablets in landscape mode */ } /* Using CSS variables with media queries */ :root { --container-width: 100%; --font-size-base: 16px; } @media (min-width: 768px) { :root { --container-width: 750px; --font-size-base: 18px; } } .container { width: var(--container-width); } body { font-size: var(--font-size-base); }
